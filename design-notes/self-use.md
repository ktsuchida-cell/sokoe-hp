# DaySelfUse セクション設計ノート（自社導入後の改善効果）

このファイルは sokoe Day LP の `DaySelfUse` セクション設計判断を記録するためのもの。
2026-05-18 オーナーフィードバックを起点に、Claude opus 3 体 + Codex の 4 視点レビューを集約した。

参考: `design-notes/lp-overall.md`、`design-notes/positioning.md`

---

## 1. セクション目的

- E-E-A-T 訴求（自社運営施設での実証）による信頼形成。
- 「机上の空論ではない、実際に sokoe Day を使うとここまで変わる」という導入後の姿を可視化する。
- LP 全体での位置づけ: Pain → Solution → Positioning → **SelfUse（ここ）** → Features → Pricing。
- 想定読者の温度: 55–65°C（共感・誤解防止が済んで、「で、本当に効くの？」を確認したい段階）。

---

## 2. 想定読者

| 役割 | 状態 |
|---|---|
| 施設長・管理者（中小規模デイ、1 日平均 20–60 名） | 「うちの施設で同じことが起きるか」を確認したい |
| 副施設長・ケアマネ | 「現場のしんどさを誰が分かってくれているか」を見たい |
| オーナー会社の経営層 | 「数字の根拠と業界知見の深さ」を確認したい |

## 4. 問題点とその診断（現状コードの 4 つの弱点）

1. **Before → After の対比がテキスト記号「→」のみ**：「30 分 → 数分」の矢印を読まないと改善幅を理解できない。スキャン時に素通りされる。
2. **5 枚カードが完全に等価**：番号バッジ + タイトル + 段落の同一構造が 5 連続。リズム・強弱がなく、「主役級」のケアプラン削減事例と「補助級」の名刺 OCR が同じ視覚的重みを持つ。
3. **After（導入後の状態）が言語的にしか提示されない**：オーナーが新しく求める「導入後の現場の絵」が無く、変化の説明だけで終わっている。eyebrow「SELF-USE PROOF」も新意図とズレている。
4. **実写代替の視覚言語が定義されていない**：写真を意図的に外した穴を埋める要素（紙バインダーアイコン / 画面モック）が無いため、テキストの説得力のみで信頼を取る構造。施設長は「実際の画面・現場のモノ」で判断したい層。

---

## 5. 4 視点レビューの合意点と分岐点

### 合意点（4 視点共通）

- 事例ごとに `before / after` の 2 構造に再設計し、視覚的対比をつくる。
- 実写代替は SVG ベースの軽量グラフィック（紙書類・バインダー・録音マイク・名刺・スマホ画面）。Before 側 = stone 系グレーで「くすませた紙世界」、After 側 = ink / brand-red 系で「現役感」。
- caption は **大見出し直下またはセクション末尾** に固定で配置し、出所と限界を先に自白する。
- 「削減」「実証」など断定調を弱める：「自社運営施設での運用体感」「目安」を併記。
- セクション末尾に CV CTA は置かない（55-65°C は信頼形成段階）。

### 分岐点と採用判断

| 分岐 | 案 | 採用 |
|---|---|---|
| 全 5 事例を Before/After にするか上位だけか | 全 5 事例を 2 カラム比較で統一 | ✅ 採用（5 事例の信頼性は同列、構造を揃えるほうが「これは効果集です」と一目で伝わる） |
| Before/After をバーチャートで見せるか 2 カラムで見せるか | 2 カラム比較（Before / 矢印 / After） | ✅ 採用（バーチャートは時間削減系には強いが、「バインダー 0 化」「数秒で営業履歴」のような非時間削減事例には合わない） |
| 事例の並び順 | 施設長の痛点頻度順に並び替え | ✅ 採用 |
| eyebrow と見出し | 「AFTER INTRODUCTION」+「自社導入後、現場はこう変わった。」 | ✅ 採用 |
| 末尾の CTA | 置かない | ✅ 採用 |
| 数字の修飾語 | 「概ね」「程度」「目安」を局所的に追加 | ✅ 採用 |

### 6.2 視覚言語（Before / After のアイコン対応表）

| 事例 | Before icon | After icon |
|---|---|---|
| 通所介護計画書 | 紙の書類 + 手書きペン | スマホ + AI スパーク |
| マシン訓練バインダー | バインダー + 付箋 | スマホ + チェックマーク |
| 個別評価コメント | 手書きペン + 評価用紙 | スマホ画面 + コメントバブル |
| 担当者会議の議事録 | マイク + 紙ノート | スマホ + 自動要約バブル |
| 名刺管理 | 名刺の束 + Excel アイコン | スマホ + 連絡先カード |

すべて `lucide-react` の既存ライブラリ範囲で表現可能（FileText, ClipboardList, Mic, CreditCard, Smartphone, Sparkles, Check 等）。

### 6.3 配色とサイズの根拠

| 要素 | 値 | 根拠 |
|---|---|---|
| Before カラム背景 | `bg-cream` または `bg-stone-50` | 紙世界を低彩度でくすませ「過去・苦痛」と読ませる |
| After カラム背景 | `bg-white` + 左ボーダー `brand-red` | 「現役感」を brand color のアクセントで強調 |
| 矢印 | `text-brand-red` 大型 | プレアテンティブ処理：色と方向で「変化」を言語非依存に伝える |
| Before / After ラベル | uppercase tracking-wider 小サイズ | ラベルとして識別、本文を邪魔しない |
| カードタイトル | `font-serif text-lg md:text-xl` brand-red | 既存トーン踏襲、視覚階層維持 |
| カード本文 | `text-[15px] md:text-base leading-[1.85]` | 既存セクションと同じ可読性レベル |

### 6.4 スマホ対応

- `md` 未満では grid を `grid-cols-1` に切り替え、Before → 矢印（縦向き）→ After の縦積みに。
- 矢印は md 以上で水平、md 未満で垂直（rotate-90 または ↓ 文字に切替）。
- Before / After のラベルを常に明示し、色だけに依存しない（A11y）。

## 7. コピーライティング（5 事例、痛点頻度順）

各事例：
- `category`: 業務カテゴリ（ラベル相当）
- `title`: カード見出し
- `before`: 改善前の現場
- `after`: 改善後の状態

### ① 通所介護計画書（ケアプラン）

- **title**: 通所介護計画書、月 10 時間が消えていた転記が、ほぼ無くなった
- **Before**: ケアマネから届く情報を、施設フォーマットへ毎月 1 人 30 分かけて転記。月 20 名規模で約 10 時間相当が消えていた。
- **After**: AI が施設フォーマットへ自動変換し、職員は内容を確認するだけ。転記そのものが業務から消えた。

### ② マシン訓練バインダー

- **title**: 利用者さんごとのバインダーが、紙ゼロに
- **Before**: 利用者さんごとにマシン設定・実施記録のバインダーを用意し、毎回出し入れ・更新・紛失対応が発生していた。
- **After**: マシン設定と完了の有無をアプリ内で管理。バインダー印刷・差し替え・保管が不要になった。

### ③ 個別評価コメント

- **title**: TUG・握力の測定値を入れるだけで、個別コメントと賞状 PDF が出る
- **Before**: TUG・握力などの結果に対し、利用者さん 1 人ずつ職員が言葉を選んでコメントを書いていた。30 名分で概ね 2 時間半。
- **After**: 測定値を入力すれば、AI が利用者さんに伝わる優しい言葉でコメント化。賞状 PDF も同時に出力される。

### ④ 担当者会議の議事録

- **title**: 1 時間の書き起こしが、10 分の最終チェックに
- **Before**: 会議後に録音を聞き直し、発言者を判別しながら書き起こすため、1 件で概ね 1 時間。会議件数が増えるほど夜の残業に積み上がっていた。
- **After**: その場で文字起こしと話者識別が走り、AI が要約まで出す。担当は要約を読んで整えるだけ。

### ⑤ 名刺管理

- **title**: 撮影 → 手入力 → ファイル化が、数秒で営業履歴に
- **Before**: もらった名刺を Excel に手入力し、後日「あの会社の誰だっけ？」を探す時間が積み重なっていた。
- **After**: スマホで撮るだけで氏名・事業所・連絡先・役職を AI OCR が抽出し、担当ご利用者リストにも自動で紐付く。

## 8. Caption 文言（確定版）

セクション末尾またはリード直下に固定：

> ※ 自社運営施設「レッツ倶楽部川西能勢口」（1 日平均約 30 名規模）での日常運用に基づく体感ベースの目安です。2026 年 4 月の稼働開始以降、施設長代理（当社代表）と職員の業務感覚に基づくもので、厳密な実測値ではありません。事業所の規模・業務手順により効果は異なります。

ポイント：
- 「実測ではない」と先に自白する（CLAUDE.md ガードレール準拠）
- 「効果は異なります」で誇大広告感を消す
- 「施設長代理（当社代表）」で代表の現役性を再強調
