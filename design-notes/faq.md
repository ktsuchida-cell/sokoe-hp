# DayFAQ セクション 設計ノート

`src/components/DayFAQ.tsx` の設計判断を記録。

---

## セクションの目的

導入前に施設長・管理者が抱く 10 個の典型的な疑問を解消する。
- 「カイポケと併用できる？」「導入期間は？」「セキュリティは？」など
- AI Overview / ChatGPT に verbatim 引用されやすい形式（質問→直接答え→補足）
- **FAQ で解消しなかった疑問を持つ読者は、最終的にお問い合わせへ送る**

---

## 2026-05-14 改修 ── 末尾 CTA を脚注 → 結論 CTA へ昇格

### 問題点（lp-overall.md 視覚階層問題の一例）

| # | 問題 | 影響 |
|---|---|---|
| F1 | 末尾の「他にご質問があれば、お気軽にお問い合わせください。」が `text-sm md:text-base text-stone`（14–16px 灰色）の脚注スタイル | FAQ で解消しなかった疑問を持つ読者を CV に運ぶ最後の橋なのに、視覚的に弱い |
| F2 | CTA がテキストリンク「お問い合わせはこちら →」（`text-base text-brand-red`） | Header / Pricing / LastCTA はすべて Button コンポーネントなのに、ここだけテキストリンク。ボタンの押し心地・存在感が弱い |
| F3 | リード文に「sokoe らしさ」がない | 代表自身が現場目線で答える、というブランド独自性を活かせていない |

### 改善内容

**リード文**
- サイズ: `text-sm md:text-base` → `text-base md:text-lg`（脚注 → 本文同等のサイズ）
- 色: `text-stone`（灰）→ `text-charcoal`（濃いめ、読みやすさ優先）
- コピー: 一文 → 二文に拡張、二文目で「代表（現役の施設長代理）が現場目線で答える」と sokoe 独自性を出す

**CTA**
- テキストリンク → `Button variant="secondary" size="md"` に昇格
- ラベル: 「お問い合わせはこちら →」→「お問い合わせ」（CTA ラベル統一の方針に従う）

### 根拠

| 階層 | 役割 | 採用サイズ |
|---|---|---|
| H2「よくあるご質問」 | セクション見出し | `text-3xl/4xl`（維持） |
| **末尾誘導文** | **FAQ で解消しなかった人への声かけ（結論レイヤー）** | **`text-base/lg`（脚注ではなく本文）** |
| **末尾 CTA** | **最後の CV ポイント** | **Button md（他 CTA と同フォーマット）** |
| Accordion 質問 | 一覧 | 維持 |
| Accordion 答え | 詳細 | 維持 |

末尾誘導文は **「FAQ で疑問が解消しなかった人を逃さない最後の橋」** という役割。これは「結論レイヤー」であり脚注ではない。`positioning.md` の結論文ロジック（注記サイズ → 結論サイズ）と同じ判断を、ここでも適用。

### 実装注意

- CTA を Button md にすることで、Header の `お問い合わせ` Button と視覚的整合性が取れる
- リード文の 2 行目「代表（現役の施設長代理）が...」は **モバイルで改行制御**（`<br className="hidden md:block" />`）
- 次セクション（DayAboutFounder）への送り出しとしても自然な距離 = `mt-14 md:mt-16`
