## 2. 制約

- 5 事例は維持（軽い文言整理は OK）
- 顔写真は使わない（オーナー方針）
- 数値訴求は実測根拠なしで断定しない（CLAUDE.md ガードレール）
- 背景 `tint-pink` 維持
- Before/After 2 カラムは復活させない（v1 で撤去済み）
- caption は維持（出所明記）
- セクション末尾の CTA（DaySelfUseCTA）は別ファイルで独立済み

## 3. 4 視点レビューの合意点

| 軸 | 合意内容 |
|---|---|
| 視覚要素 | **lucide-react アイコン**で識別性を上げる（5 事例で異なる業務アイコン） |
| 文字量 | タイトル 15-25 字／本文 40-50 字へ圧縮（現状の半分） |
| レイアウト | PC 2 カラム / スマホ 1 カラム |
| 業務識別 | 「通所介護計画書」「マシン訓練」など **業務カテゴリタグ**を末尾に小さく |
| 視覚階層 | 背景 `tint-pink` → 白カード → After 側 `brand-red` アクセントの 3 層 |
| 視線起点 | アイコンを左上または中央上に置き、「読み始め点」を 1 つに集約 |
| 区切り | タイトルと本文の間に hairline（細い罫線）で視覚的に分節 |

## 4. 分岐点と採用判断

| 分岐 | 案 | 採用 |
|---|---|---|
| 5 枚を 1 カラム維持か 2 カラムグリッドか | 2 カラム（5 枚目は span-2 で中央） | ✅ |
| Before/After を復活させるか | 復活させない（v1 オーナー指示尊重） | ✅ |

| アイコン色 | brand-red 系の薄塗り円形バッジ | ✅ |

## 5. 改善案：採用設計

### 5.1 データモデル

```ts
type Improvement = {
  icon: LucideIcon;       // 業務アイコン
  categoryTag: string;    // 業務カテゴリ（小タグ）
  title: string;          // 結論一行（15-25 字）
  body: string;           // 状態説明（40-50 字）
};
```

### 5.2 各事例（痛点頻度順を維持）

| # | icon | categoryTag | title | body |
|---|---|---|---|---|
| 1 | FileText | 通所介護計画書 | ケアプラン作成が、転記ゼロに。 | ケアマネから届くケアプランを、アプリで自動作成。月 20 名規模で 10 時間相当の転記が、ほぼなくなりました。 |
| 2 | FolderOpen | マシン訓練 | マシン訓練のバインダーが、紙ゼロに。 | マシン設定も実施記録もアプリ内で完結。利用者さんごとに紙のバインダーを出す作業がなくなりました。 |
| 3 | Activity | 個別評価 | 測定値を入れるだけで、個別コメント生成。 | TUG・握力の測定値を入力すれば、AI が優しい言葉でコメント化。30 名分の手書き作業が、入力だけで完了します。 |
| 4 | Mic | 担当者会議 | 議事録が、10 分の最終チェックに。 | 録音をその場で文字起こし、AI が要約と発言者識別。1 件 1 時間の書き起こしが、要約のチェックだけに。 |
| 5 | Contact | ケアマネ営業 | 名刺が、撮影だけで営業履歴に。 | スマホで撮るだけで AI OCR が氏名・事業所・連絡先・役職を抽出。Excel への手入力作業がなくなりました。 |

### 5.3 カード雛形（1 枚）

```
+-----------------------------------------------------------+
|  ┌────┐                                                    |
|  │icon│ <- brand-red 薄塗り円バッジ 48px                   |
|  └────┘                                                    |
|                                                            |
|  ケアプラン作成が、転記ゼロに。  <- title (serif, brand-red)
|                                                            |
|  ────────                       <- hairline (border-border)
|                                                            |
|  ケアマネから届くケアプランを、  <- body (stone, 15px)
|  アプリで自動作成。月 20 名規模  
|  で 10 時間相当の転記が、ほぼ   
|  なくなりました。                
|                                                            |
|  通所介護計画書                  <- categoryTag (uppercase, mid, xs)
+-----------------------------------------------------------+
```

### 5.4 配色とサイズの根拠

| 要素 | 値 | 根拠 |
|---|---|---|
| カード背景 | `bg-white` | tint-pink 背景の上で「白カードがフォーカス領域」と読ませる |
| カード枠 | `border border-border` | 弱い枠で囲み感だけ確保 |
| アイコンバッジ | `bg-brand-red/10 text-brand-red` 円形 48px | brand-red の薄塗りで親近感、アクセントを連打しない |
| タイトル | `font-serif text-base md:text-lg font-bold text-brand-red` | 既存 SelfUse トーン踏襲、視覚階層維持 |
| hairline | `border-t border-border` | タイトルと本文を視覚的に分節し、スキャン時にタイトルだけ拾いやすく |
| body | `text-stone text-[14px] md:text-[15px] leading-[1.85]` | 既存可読性レベルを維持しつつフォントは控えめ |
| categoryTag | `text-[11px] uppercase tracking-[0.14em] text-mid` | 小さいラベルで「何の業務か」を 0.5 秒で識別 |

### 5.5 レイアウト（レスポンシブ）

```
mobile (sm 未満):  grid-cols-1
md 以上:           grid-cols-2
最後の 5 枚目:      md:col-span-2 → md:max-w-1/2 mx-auto で中央配置
```

5 番目だけ最終行で 1 列になるので、`md:col-span-2` + 中央寄せで「孤立感」を消す。
