# DayPositioning セクション 設計ノート

このファイルは `src/components/DayPositioning.tsx` の設計判断を記録するためのもの。
変更を加えるたびに「**なぜ**そのサイズ・配置にしたか」をここに残す。

---

## セクションの目的

施設長・管理者が「sokoe Day はカイポケ等の代替か？」と誤解しないようにする。
- 左カード: 介護記録ソフト（カイポケ・ナーシングネット 等）= 請求業務担当 → そのまま継続
- 右カード: sokoe Day = 現場業務担当 → 追加で導入
- **関係性は「対立」ではなく「プラス（加算）」**

## 2026-05-14 改修（v2）── プラス関係表現と結論文の昇格

### 問題点

| # | 問題 | 影響 |
|---|---|---|
| P1 | カード 2 枚が単純な `grid-cols-2` で並び、関係性が「並列」にしか見えない | 「乗り換え／代替」と誤読されるリスク |
| P2 | 右カードの「ここを SOKOE DAY が」バッジが冗長（カード内の "現場は、sokoe Day が" と二重発話） | バッジが情報を運ばない、視覚ノイズ |
| P3 | 末尾コピーが `text-[13px] text-mid` の脚注スタイルだが、内容は「請求 = 既存、現場 = sokoe Day」というセクションの結論文 | 視覚階層と意味階層がねじれ、最重要メッセージが最も読まれない位置に置かれる |
| P4 | 右カードだけ `border-2 / bg-tint-pink / scale-[1.02] / shadow` で過剰強調 | 「役割分担」のメッセージと矛盾、「sokoe Day が勝つ」絵に見える |

### 改修 A: カード間「＋」関係表現

**Desktop**: `grid-cols-[1fr_auto_1fr]` で「カード + プラス + カード」の数式的配置。プラスは円形チップ（直径 56-64px）、`bg-white / border-border / text-brand-red text-[28-32px] font-light`。

**Mobile**: `grid-cols-1` の中に「カード → プラスチップ → カード」を縦に並べる。プラスチップは小さめ（直径 44px）で `-my-1` で少しカードに重ねて「つなぐ」読みに。

**根拠**:
- 円形チップにすることで「カードの延長」ではなく**独立した算術記号**として読まれる
- `font-light` の `+` は太字 UI ボタンと差別化される
- `text-brand-red` で「加わるのは sokoe Day」のニュアンスを弱く出す
- `items-stretch` 親 grid のままで、`+` セルだけ `items-center` で縦中央配置

### 改修 B: カード強調差をやわらげる

右カードの `border-2 / scale-[1.02]` を撤去し、左右を**同寸・同枠線太さ（1px）**に揃える。差別化は背景色（左 `bg-bg-muted` / 右 `bg-tint-pink/40`）とラベル文字色（右 `text-brand-red`）のみ。

### 改修 C: バッジ「ここを SOKOE DAY が」を削除

カード内の "現場は、sokoe Day が" eyebrow ラベルだけで十分。バッジは二重発話だったので撤去。「+」アイコンが「sokoe Day をプラスする」意味を運ぶので、別途バッジは不要。

### 改修 D: 末尾コピーを「結論文」として昇格

| 階層 | 役割 | size | 根拠 |
|---|---|---|---|
| H2 (見出し) | セクションの問い | `text-3xl/4xl` (≈28-36px) | 維持 |
| 本文 (リード) | 説明 | `text-base/lg` (16-18px) | 維持 |
| **結論文** | **答え（最重要）** | **`text-[20-26px]` serif text-ink** | **本文より一段大きく** |
| カード内文字 | 詳細 | 13-15px | 維持 |

**理由**:
- このコピーは「**請求はソフトで、現場は sokoe Day で**」というセクションのコアメッセージそのもの（注記ではない）
- 視覚階層理論では「結論文は H2 より小さく、本文より大きい」が原則
- 現状の `text-[13px] text-mid` は階層逆転だった
- `font-serif` を当てることで H2 と対をなす一文だと暗黙に伝わる
- `text-ink`（最濃色）で断定の強さを出す
- 上部に `CONCLUSION` eyebrow ラベル + 下部に短い装飾ライン（`h-px w-12 bg-brand-red/40`）で結論を視覚的に「囲い込む」

### 実装上の注意

- **アクセシビリティ**: `+` チップは装飾なので `aria-hidden="true"`。ただしスクリーンリーダー向けに `sr-only` で関係性を補う検討。
- **モバイル `<br>` 分岐**: 結論文の改行はモバイル/デスクトップで分ける。
- **次セクションとの間隔**: `mt-14 md:mt-16` 程度（広げすぎるとフッターっぽく見える）。
